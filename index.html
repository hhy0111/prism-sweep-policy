This page exists so the in-game consent flow has a local, reviewable privacy target during development.

Data Used In This Prototype

  • Local save data such as sound settings, ad consent state, and reward claim state.
  • Advertising identifiers and ad event data when a live ad SDK is enabled in a native build.
  • Basic device and session diagnostics needed for gameplay stability and ad delivery.

Advertising

This game prototype is designed to support rewarded ads, interstitial ads, and app-open ads. In development builds, web runtime uses mock ads for debugging. Native production builds must clearly explain what ad partners collect and how consent is handled.
